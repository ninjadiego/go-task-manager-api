# Contributing to Go Task Manager API

Thank you for your interest in contributing! We welcome pull requests, issue reports and ideas.

## Development Workflow

1. Fork the repository and clone your fork.
2. 2. Create a feature branch from main: git checkout -b feat/my-feature
   3. 3. Run make fmt && make lint && make test before committing.
      4. 4. Use Conventional Commits (feat:, fix:, docs:, chore:).
         5. 5. Open a pull request with a clear description.
           
            6. ## Code Style
           
            7. - Follow standard Go conventions and gofmt formatting.
               - - Keep functions small and focused.
                 - - Add unit tests for any new business logic.
                   - - Document exported identifiers with Go doc comments.
                    
                     - ## Reporting Bugs
                    
                     - When filing an issue, please include steps to reproduce, expected vs actual behavior, Go version and OS.
